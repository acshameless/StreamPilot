# AI 直播提词助手 V1 PRD

> 极简版：一个能离线跑的、单设备本地大屏 SKU 提词卡片。

---

## 1. 产品目标

给单个主播一个本地大屏 SKU 提词器，能记 SKU 信息、卡片式切换、专心带货。

不做实时同步、不做 ASR、不做敏感词、不做账号体系。

---

## 2. V1 范围

### ✅ 必做

- 创建 / 编辑 / 删除 SKU
- 大屏提词模式（全屏 + 大字体 + 深色）
- 卡片式 SKU 切换（左右键 / 点击 / 空格）
- 数据本地持久化（LocalStorage）

### ❌ 不做（留 V2+）

- 房间多人协作
- 双设备同步
- ASR 实时字幕
- 敏感词高亮告警
- 用户登录 / 多账号
- 云端同步

---

## 3. 用户角色

只有一个角色：**主播**（自播自控）。

---

## 4. 页面设计

### 4.1 首页 `/`

主页 = SKU 列表 + 入口。

**UI 要素**

- 顶部：标题「直播提词助手」 + 「+ 新增 SKU」按钮 + 「进入大屏」按钮
- 中部：SKU 卡片列表（默认 3 列网格，移动端单列）
  - 每张卡片显示：商品名、价格、卖点条数
  - 卡片右下角：编辑、删除按钮
- 空状态：「还没有 SKU，点这里新增第一个商品 →」CTA

**操作**

| 操作 | 入口 |
| --- | --- |
| 新增 SKU | 顶部按钮 → 弹出表单 |
| 编辑 SKU | 卡片「编辑」按钮 |
| 删除 SKU | 卡片「删除」按钮（二次确认） |
| 进入大屏 | 顶部按钮 → 路由 `/screen` |

---

### 4.2 SKU 编辑表单

入口：弹窗（推荐）或独立页 `/sku/[id]/edit`。

**字段**

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| 商品名称 | ✅ | string | 单行输入 |
| 价格 | ✅ | string | 例 "129"（纯字符串，避免单位耦合） |
| 材质 | | text | 多行 |
| 卖点 | | string[] | 动态添加列表（每行一条） |
| 禁说词 | | string[] | 动态添加列表（V1 仅展示，不告警） |

---

### 4.3 大屏提词页 `/screen`

**入屏行为**

- 默认显示第一个 SKU（或上次退出时的索引）
- 自动尝试触发 fullscreen API（用户首次需手势激活，提供「全屏」按钮兜底）
- 深色背景（`#0a0a0a`），白色 / 高亮色文字

**布局（自上而下）**

```
┌───────────────────────────────────────────┐
│  [4 / 12]         冰感长裤        ¥ 129   │ ← 顶部条：序号、商品名、价格
├───────────────────────────────────────────┤
│                                           │
│   材质                                    │
│   65% 棉  35% 聚酯纤维                    │
│                                           │
│   卖点                                    │
│   • 凉感                                  │
│   • 显瘦                                  │
│   • 高弹                                  │
│                                           │
│   ⚠ 禁说                                  │
│   纯棉  /  真丝  /  100%                  │ ← 红色或橙色加粗
│                                           │
└───────────────────────────────────────────┘
   ←  上一个                   下一个  →
```

**字号要求（默认 1080p / 距离 1m 阅读）**

| 元素 | 最小字号 |
| --- | --- |
| 商品名 / 价格 | ≥ 64px |
| 卖点 / 材质正文 | ≥ 40px |
| 禁说词 | ≥ 36px，红色加粗 |
| 顶部序号 | ≥ 28px |

**操作**

| 操作 | 触发方式 |
| --- | --- |
| 下一个 SKU | 右箭头 / 空格 / 屏幕右半边点击 |
| 上一个 SKU | 左箭头 / 屏幕左半边点击 |
| 退出大屏 | ESC |
| 返回主页 | 顶部「←」（鼠标移入大屏时显示） |
| 切换全屏 | 顶部「全屏」按钮 |

## 5. 数据结构

### LocalStorage Keys

| Key | 内容 |
| --- | --- |
| `zhibo:skus:v1` | SKU 数组 |
| `zhibo:screen:lastIndex` | 大屏最后一次显示的 SKU 索引 |

### SKU 类型

```ts
interface SKU {
  id: string;            // crypto.randomUUID()
  name: string;
  price: string;
  material: string;
  sellingPoints: string[];
  bannedWords: string[];
  createdAt: number;     // Date.now()
  updatedAt: number;
}
```

## 6. 交互细节

### 6.1 大屏切换防抖

连续按方向键不要触发动画堆积，用 transition + lock，切换间隔 ≤ 100ms 视为同一次。

### 6.2 切换动画

卡片左右滑入（200ms ease-out），不要花哨效果。

### 6.3 空状态保护

没有 SKU 时进入大屏 → 显示「请先在主页添加 SKU」+「返回」按钮，不允许空状态全屏。

### 6.4 全屏 API

- 进大屏页时 `element.requestFullscreen()`
- 失败（如 Safari 限制）则保持窗口模式
- 提供顶部「全屏」按钮兜底

### 6.5 长内容自滚动（次要）

单个 SKU 内容超过一屏时，正文区域可上下滚动条（鼠标滚轮 / 触屏滑动），V1 不做自动滚动。

## 8. 验收标准

| # | 验收点 |
| --- | --- |
| 1 | 新增 SKU 后刷新页面仍存在 |
| 2 | 大屏页可全屏，方向键和点击都能切换 |
| 3 | 商品名 / 价格 / 材质 / 卖点 / 禁说词都能正确展示 |
| 4 | 单 SKU 切换 P95 视觉延迟 < 100ms（本地） |
| 5 | Lighthouse Performance ≥ 90 |
| 6 | 部署到 Vercel 后公网 URL 可访问 |
| 7 | Chrome / Safari / Edge 桌面端正常 |
